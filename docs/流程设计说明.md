# 企业管理系统 - 流程模块设计说明

## 一、概述

### 1.1 文档目的
本文档旨在详细说明企业管理系统中**项目申请**、**出差申请**、**自定义流程**三大核心流程模块的设计方案，为开发团队提供清晰的实现指导。

### 1.2 适用范围
- 项目申请管理模块
- 出差申请管理模块
- 自定义流程引擎模块

### 1.3 术语定义
| 术语 | 说明 |
|------|------|
| 流程实例 | 一次具体的申请流程，包含申请数据、审批路径、状态等信息 |
| 审批节点 | 流程中的审批环节，可配置审批人、审批条件等 |
| 流程模板 | 预定义的流程结构，包含节点配置、流转规则等 |
| 业务数据 | 申请表单中填写的具体业务信息 |

### 3.2 员工角色与审批身份整合

#### 3.2.1 员工角色体系
基于现有员工管理模块，定义以下审批角色：

```typescript
// 员工角色枚举（与现有系统兼容）
enum EmployeeRole {
  // 基础角色
  EMPLOYEE = 'employee',           // 普通员工
  DEPT_MANAGER = 'dept_manager',   // 部门经理
  FINANCE = 'finance',             // 财务人员
  HR = 'hr',                       // 人力资源
  ADMIN = 'admin',                 // 系统管理员
  
  // 高级审批角色
  DIRECTOR = 'director',           // 总监
  VP = 'vp',                       // 副总裁
  GM = 'gm',                       // 总经理
  CEO = 'ceo',                     // CEO
  
  // 特殊角色
  PROJECT_MANAGER = 'pm',          // 项目经理
  TECH_LEAD = 'tech_lead',         // 技术负责人
  FINANCE_MANAGER = 'finance_manager' // 财务经理
}

// 员工信息扩展（基于现有Employee模型）
interface Employee {
  id: string;
  name: string;
  department: string;
  position: string;
  email: string;
  phone: string;
  
  // 审批相关字段（新增）
  roles: EmployeeRole[];            // 员工可拥有多个角色
  approverLevel: number;            // 审批级别（1-10）
  canApproveTypes: string[];        // 可审批的申请类型
  
  // 组织架构
  managerId?: string;               // 直属上级ID
  subordinateIds?: string[];        // 下属ID列表
}
```

#### 3.2.2 审批人解析策略

```typescript
// 审批人解析器（将角色/规则解析为具体员工）
class ApproverResolver {
  
  // 根据策略解析审批人
  async resolveApprovers(
    strategy: ApproverStrategy, 
    context: ApprovalContext
  ): Promise<Employee[]> {
    
    switch (strategy.type) {
      case 'user':
        // 直接指定员工
        return await this.getEmployeesByIds(strategy.userIds);
        
      case 'role':
        // 按角色查找
        return await this.getEmployeesByRole(
          strategy.role, 
          context.applicant?.department
        );
        
      case 'dept_leader':
        // 部门负责人
        return await this.getDeptLeader(context.applicant?.department);
        
      case 'manager':
        // 直属上级
        return await this.getManager(context.applicant?.id);
        
      case 'hierarchy':
        // 按组织架构层级
        return await this.getApproversByHierarchy(
          context.applicant?.id,
          strategy.level
        );
        
      case 'self_select':
        // 申请人自选
        return []; // 前端展示选择器
        
      default:
        return [];
    }
  }
  
  // 获取部门负责人
  private async getDeptLeader(department: string): Promise<Employee[]> {
    // 从员工表中查询部门经理
    const response = await getEmployees();
    return response.data.filter(emp => 
      emp.department === department && 
      emp.roles?.includes(EmployeeRole.DEPT_MANAGER)
    );
  }
  
  // 获取直属上级
  private async getManager(employeeId: string): Promise<Employee[]> {
    const employee = await this.getEmployeeById(employeeId);
    if (employee?.managerId) {
      const manager = await this.getEmployeeById(employee.managerId);
      return manager ? [manager] : [];
    }
    return [];
  }
}
```

#### 3.3.3 基于员工角色的审批流程

```
┌─────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  提交   │ → │ 部门经理    │ → │ 财务经理    │ → │ 总经理      │
│  申请   │    │ (自动分配)  │    │ (自动分配)  │    │ (自动分配)  │
└─────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                      │
                                                 ┌─────────┐
                                                 │  立项   │
                                                 │  完成   │
                                                 └─────────┘
```

**审批人自动分配规则**:

| 审批节点 | 分配规则 | 员工角色 |
|----------|----------|----------|
| 部门经理审批 | 申请人所属部门的部门经理 | `DEPT_MANAGER` |
| 财务审批 | 财务部门经理 | `FINANCE_MANAGER` |
| 总经理审批 | 公司最高审批人 | `GM` 或 `CEO` |

**条件分支（基于预算金额）**:
- 预算 < 10万: 部门经理 → 总经理（跳过财务）
- 预算 10-50万: 部门经理 → 财务经理 → 总经理
- 预算 > 50万: 部门经理 → 财务经理 → 总经理 → 董事会

#### 3.4.3 基于组织架构的审批流程

```
┌─────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  提交   │ → │ 直接主管    │ → │ 部门经理    │ → │ 财务审核    │
│  申请   │    │ (直属上级)  │    │ (自动分配)  │    │ (高费用)    │
└─────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                           │
                                                      ┌─────────┐
                                                      │  出差   │
                                                      │  批准   │
                                                      └─────────┘
```

**审批人自动分配规则**:

| 审批节点 | 分配规则 | 员工关系 |
|----------|----------|----------|
| 直接主管审批 | 申请人的直属上级 | `managerId` 字段关联 |
| 部门经理审批 | 申请人所属部门的部门经理 | `DEPT_MANAGER` 角色 |
| 财务审核 | 财务部门（费用>5000时触发） | `FINANCE` 角色 |

**特殊处理**:
- **紧急出差**: 可勾选"紧急"标记，发送即时通知给所有审批人
- **国外出差**: 自动增加"外事审批"节点（`HR` 角色）
- **多人同行**: 支持批量申请，同行人员只需确认

#### 5.2.2 流程节点（Process Node）
```typescript
interface ProcessNode {
  id: string;
  type: NodeType;
  name: string;
  config: NodeConfig;

interface ApprovalNodeConfig {

  timeoutHours?: number;    // 超时时间
  reminderHours?: number;   // 提醒时间

// 多审批节点配置（支持一对多、多对一场景）
interface MultiApprovalNodeConfig {
  approverType: 'user' | 'role' | 'dept_leader' | 'self_select';
  approvers: string[];
  mode: 'one_to_many' | 'many_to_one' | 'many_to_many';
  // 一对多：一个节点分发到多个审批人并行处理
  // 多对一：多个前置节点汇聚到一个审批人
  // 多对多：多个前置节点分发到多个审批人
  
  // 审批策略
  strategy: {
    type: 'all' | 'any' | 'sequential' | 'percentage' | 'weight';
    requiredCount?: number;       // 需通过人数
    requiredPercentage?: number;  // 需通过百分比
    weights?: Record<string, number>; // 权重配置（key为审批人ID）
  };
  
  // 汇聚配置（多对一时使用）
  joinConfig?: {
    waitAll: boolean;             // 是否等待所有前置节点完成
    timeoutJoin: boolean;         // 超时是否自动汇聚
    timeoutHours?: number;        // 汇聚超时时间
  };
  
  // 分发配置（一对多时使用）
  distributeConfig?: {
    autoDistribute: boolean;      // 自动分发

  };
  
  allowTransfer: boolean;
  allowAddApprover: boolean;
  timeoutHours?: number;
  reminderHours?: number;
}

// 并行网关配置（一对多）
interface ParallelGatewayConfig {
  gatewayType: 'fork' | 'join';
  // fork: 分发到多个分支（一对多）
  // join: 汇聚多个分支（多对一）
  
  // 分支条件（fork时使用）
  branchConditions?: {
    branchId: string;
    condition?: Condition;  // 为空表示无条件执行
  }[];
  
  // 汇聚策略（join时使用）
  joinStrategy?: 'all' | 'any' | 'n_of_m';
  requiredBranches?: number;  // n_of_m时需要汇聚的分支数
  
  // 超时配置
  timeoutConfig?: {
    enabled: boolean;
    timeoutHours: number;
    autoComplete: boolean;    // 超时自动完成
  };
}

// 包容网关配置（条件分支一对多）
interface InclusiveGatewayConfig {
  gatewayType: 'fork' | 'join';
  
  // 分支条件（fork时使用，可执行多个分支）
  branches: {
    branchId: string;
    condition: Condition;
    default?: boolean;  // 默认分支（其他条件都不满足时执行）
  }[];
  
  // 汇聚策略（join时使用）
  joinStrategy: 'all_completed' | 'any_completed' | 'specific';
  requiredBranchIds?: string[];  // specific时需要汇聚的分支
  
  // 完成条件
  completionCondition?: {
    type: 'all' | 'count' | 'percentage';
    value?: number;
  };
}

interface ConditionNodeConfig {
  conditions: Condition[];
}

interface Condition {
  field: string;
  operator: 'eq' | 'ne' | 'gt' | 'gte' | 'lt' | 'lte' | 'in' | 'contains';
  value: any;
  logic: 'and' | 'or';
}
```

### 5.3 多节点流程模式设计

#### 5.3.1 一对一流程（串行审批）

**场景**: 最简单的线性审批流程，每个节点只有一个审批人，依次执行。

```
┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐
│开始 │ → │审批A│ → │审批B│ → │审批C│ → 结束
└─────┘    └─────┘    └─────┘    └─────┘
```

**特点**:
- 节点类型: `approval`
- 审批模式: `sequential`（顺序签）
- 每个节点只有一个审批人
- 前一节点完成后才进入下一节点

**配置示例**:
```typescript
{
  id: 'node_1',
  type: 'approval',
  name: '部门经理审批',
  config: {
    approverType: 'user',
    approvers: ['manager_001'],
    approvalMode: 'sequential',
    allowTransfer: true,
    timeoutHours: 24
  }
}
```

#### 5.3.2 一对多流程（并行审批）

**场景**: 一个节点需要多个审批人同时审批，任一通过或全部通过才进入下一节点。

**模式A: 或签（任一通过）**
```
┌─────┐    ┌─────┐    ┌─────┐
│开始 │ → │审批A│ → │结束 │
└─────┘    ├──┬──┤    └─────┘
           ↓  ↓
        ┌──┐┌──┐
        │A1││A2│  （A1或A2任一通过即可）
        └──┘└──┘
```

**模式B: 会签（全部通过）**
```
┌─────┐    ┌─────┐    ┌─────┐
│开始 │ → │审批B│ → │结束 │
└─────┘    ├──┬──┤    └─────┘
           ↓  ↓
        ┌──┐┌──┐
        │B1││B2│  （B1和B2都通过才行）
        └──┘└──┘
```

**模式C: 比例签（按比例通过）**
```
┌─────┐    ┌─────┐    ┌─────┐
│开始 │ → │审批C│ → │结束 │
└─────┘    ├──┬──┤    └─────┘
           ↓  ↓
        ┌──┐┌──┐
        │C1││C2│  （需50%以上通过）
        └──┘└──┘
```

**特点**:
- 节点类型: `multi_approval`
- 模式: `one_to_many`
- 支持多种审批策略: `any` | `all` | `percentage`

**配置示例**:
```typescript
{
  id: 'node_2',
  type: 'multi_approval',
  name: '多人会签',
  config: {
    approverType: 'role',
    approvers: ['role_manager', 'role_director'],
    mode: 'one_to_many',
    strategy: {
      type: 'percentage',
      requiredPercentage: 60  // 60%以上通过即可
    },
    distributeConfig: {
      autoDistribute: true,
      distributeRule: 'round_robin'
    }
  }
}
```

#### 5.3.3 多对一流程（汇聚审批）

**场景**: 多个前置分支完成后，汇聚到一个节点进行统一审批。

```
         ┌─────┐
    ┌──→ │分支A│ ──┐
    │    └─────┘   │
┌───┤              ↓    ┌─────┐    ┌─────┐
│开始│              ├→ │汇聚 │ → │结束 │
└───┤              ↑    └─────┘    └─────┘
    │    ┌─────┐   │
    └──→ │分支B│ ──┘
         └─────┘
```

**汇聚策略**:

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| `all` | 等待所有分支完成 | 所有分支都必须完成 |
| `any` | 任一分支完成即可 | 最快路径优先 |
| `n_of_m` | 指定数量分支完成 | 部分分支完成即可 |

**特点**:
- 节点类型: `parallel_join` 或 `inclusive_join`
- 模式: `many_to_one`
- 支持超时自动汇聚

**配置示例**:
```typescript
// 并行汇聚节点
{
  id: 'join_node_1',
  type: 'parallel_join',
  name: '并行汇聚',
  config: {
    gatewayType: 'join',
    joinStrategy: 'all',  // 等待所有分支
    timeoutConfig: {
      enabled: true,
      timeoutHours: 48,
      autoComplete: true  // 超时自动完成
    }
  }
}

// 多审批汇聚节点
{
  id: 'node_3',
  type: 'multi_approval',
  name: '汇总审批',
  config: {
    approverType: 'user',
    approvers: ['ceo'],
    mode: 'many_to_one',
    joinConfig: {
      waitAll: true,
      timeoutJoin: true,
      timeoutHours: 72
    }
  }
}
```

```
         ┌─────┐         ┌─────┐

    │    └─────┘   │    └─────┘

                      └─────┘
```

#### 5.3.5 条件分支一对多（包容网关）

**场景**: 根据条件动态决定执行哪些分支，可同时执行多个分支。

```
                    ┌─────────┐
               ┌──→ │财务审批 │ (金额>10000)
               │    └─────────┘
               │
┌─────┐    ┌───┴───┐    ┌─────────┐
│开始 │ → │条件分支│ → │技术审批 │ (类型=研发)
└─────┘    └───┬───┘    └─────────┘
               │
               │    ┌─────────┐
               └──→ │经理审批 │ (默认分支)
                    └─────────┘
```

**特点**:
- 节点类型: `inclusive` (fork) / `inclusive_join` (join)
- 可执行多个满足条件的分支
- 汇聚时等待所有激活的分支

### 5.5 多节点流程引擎核心算法

#### 5.5.1 流程推进引擎（支持多节点）

```typescript
class MultiNodeProcessEngine {
  
  // 启动流程实例
  async startInstance(definitionId: string, businessData: any): Promise<ProcessInstance> {
    const definition = await this.getDefinition(definitionId);
    const instance = this.createInstance(definition, businessData);
    
    // 找到开始节点
    const startNode = definition.nodes.find(n => n.type === 'start');
    
    // 推进到第一个任务节点
    await this.advance(instance, startNode.id);
    
    return instance;
  }
  
  // 核心推进方法（支持多节点场景）
  async advance(instance: ProcessInstance, currentNodeId: string): Promise<void> {
    const definition = await this.getDefinition(instance.definitionId);
    const currentNode = definition.nodes.find(n => n.id === currentNodeId);
    
    // 根据节点类型处理
    switch (currentNode.type) {
      case 'approval':
        // 一对一审批节点
        await this.handleApprovalNode(instance, currentNode);
        break;
        
      case 'multi_approval':
        // 多审批节点（一对多/多对一/多对多）
        await this.handleMultiApprovalNode(instance, currentNode);
        break;
        
      case 'parallel':
        // 并行网关（一对多）
        await this.handleParallelFork(instance, currentNode);
        break;
        
      case 'parallel_join':
        // 并行汇聚（多对一）
        await this.handleParallelJoin(instance, currentNode);
        break;
        
      case 'inclusive':
        // 包容网关（条件分支一对多）
        await this.handleInclusiveFork(instance, currentNode);
        break;
        
      case 'inclusive_join':
        // 包容汇聚（多对一）
        await this.handleInclusiveJoin(instance, currentNode);
        break;
        
      case 'end':
        await this.completeInstance(instance);
        break;
    }
  }
  
  // 处理多审批节点（一对多/多对一/多对多）
  async handleMultiApprovalNode(instance: ProcessInstance, node: ProcessNode): Promise<void> {
    const config = node.config as MultiApprovalNodeConfig;
    
    switch (config.mode) {
      case 'one_to_many':
        // 一对多：分发到多个审批人
        await this.distributeToMany(instance, node, config);
        break;
        
      case 'many_to_one':
        // 多对一：汇聚后统一审批
        await this.joinToOne(instance, node, config);
        break;
        
      case 'many_to_many':
        // 多对多：复杂并行
        await this.manyToMany(instance, node, config);
        break;
    }
  }
  
  // 一对多分发处理
  async distributeToMany(instance: ProcessInstance, node: ProcessNode, config: MultiApprovalNodeConfig): Promise<void> {
    const approvers = await this.resolveApprovers(config);
    
    // 创建子任务列表
    const subTasks: Task[] = [];
    
    for (const approver of approvers) {
      const task = await this.createSubTask(instance, node, approver);
      subTasks.push(task);
    }
    
    // 记录当前激活的多审批任务
    instance.activeMultiTasks = {
      nodeId: node.id,
      tasks: subTasks.map(t => t.id),
      strategy: config.strategy,
      completedTasks: []
    };
    
    // 发送通知
    for (const task of subTasks) {
      await this.sendNotification(task.assignee, 'NEW_TASK', task);
    }
  }
  
  // 处理多审批任务完成
  async handleMultiTaskComplete(instance: ProcessInstance, taskId: string, action: Action): Promise<void> {
    const multiTaskInfo = instance.activeMultiTasks;
    const task = await this.getTask(taskId);
    
    // 记录完成的任务
    multiTaskInfo.completedTasks.push({
      taskId,
      action,
      assignee: task.assignee,
      completedAt: new Date()
    });
    
    // 根据策略判断是否继续
    const shouldContinue = await this.evaluateMultiTaskStrategy(
      multiTaskInfo,
      instance.variables
    );
    
    if (shouldContinue) {
      // 清理未完成的任务
      await this.cancelRemainingTasks(multiTaskInfo.tasks, taskId);
      
      // 推进到下一节点
      await this.advanceFromNode(instance, multiTaskInfo.nodeId);
    }
  }
  
  // 评估多任务策略
  async evaluateMultiTaskStrategy(multiTaskInfo: ActiveMultiTasks, variables: any): Promise<boolean> {
    const { strategy, completedTasks, tasks } = multiTaskInfo;
    const totalCount = tasks.length;
    const completedCount = completedTasks.length;
    const approvedCount = completedTasks.filter(t => t.action === 'agree').length;
    const rejectedCount = completedTasks.filter(t => t.action === 'reject').length;
    
    switch (strategy.type) {
      case 'any':
        // 任一通过即可
        return approvedCount >= 1;
        
      case 'all':
        // 全部通过
        return approvedCount === totalCount;
        
      case 'sequential':
        // 顺序签，按顺序完成
        return completedCount === totalCount;
        
      case 'percentage':
        // 按比例通过
        const percentage = (approvedCount / totalCount) * 100;
        return percentage >= strategy.requiredPercentage;
        
      case 'weight':
        // 按权重计算
        let totalWeight = 0;
        let approvedWeight = 0;
        
        for (const task of completedTasks) {
          const weight = strategy.weights[task.assignee.id] || 0;
          totalWeight += weight;
          if (task.action === 'agree') {
            approvedWeight += weight;
          }
        }
        
        return totalWeight > 0 && (approvedWeight / totalWeight) >= 0.5;
        
      default:
        return false;
    }
  }
  
  // 处理并行网关（一对多）
  async handleParallelFork(instance: ProcessInstance, node: ProcessNode): Promise<void> {
    const config = node.config as ParallelGatewayConfig;
    const definition = await this.getDefinition(instance.definitionId);
    
    // 获取所有出边
    const outgoingEdges = definition.edges.filter(e => e.source === node.id);
    
    // 激活所有分支
    for (const edge of outgoingEdges) {
      const targetNode = definition.nodes.find(n => n.id === edge.target);
      
      // 检查分支条件
      if (!edge.condition || await this.evaluateCondition(edge.condition, instance.variables)) {
        // 创建分支上下文
        await this.createBranchContext(instance, node.id, targetNode.id);
        
        // 推进分支
        await this.advance(instance, targetNode.id);
      }
    }
  }
  
  // 处理并行汇聚（多对一）
  async handleParallelJoin(instance: ProcessInstance, node: ProcessNode): Promise<void> {
    const config = node.config as ParallelGatewayConfig;
    const definition = await this.getDefinition(instance.definitionId);
    
    // 获取所有入边（前置分支）
    const incomingEdges = definition.edges.filter(e => e.target === node.id);
    const branchIds = incomingEdges.map(e => e.source);
    
    // 检查已完成的 branches
    const completedBranches = await this.getCompletedBranches(instance, node.id);
    
    // 根据汇聚策略判断是否继续
    let shouldJoin = false;
    
    switch (config.joinStrategy) {
      case 'all':
        shouldJoin = completedBranches.length === branchIds.length;
        break;
      case 'any':
        shouldJoin = completedBranches.length >= 1;
        break;
      case 'n_of_m':
        shouldJoin = completedBranches.length >= config.requiredBranches;
        break;
    }
    
    if (shouldJoin) {
      // 清理分支上下文
      await this.cleanupBranchContexts(instance, node.id);
      
      // 推进到下一节点
      const outgoingEdges = definition.edges.filter(e => e.source === node.id);
      for (const edge of outgoingEdges) {
        await this.advance(instance, edge.target);
      }
    }
  }
  
  // 处理包容网关（条件分支一对多）
  async handleInclusiveFork(instance: ProcessInstance, node: ProcessNode): Promise<void> {
    const config = node.config as InclusiveGatewayConfig;
    const definition = await this.getDefinition(instance.definitionId);
    
    let hasActivatedBranch = false;
    const activatedBranches: string[] = [];
    
    // 评估所有分支条件
    for (const branch of config.branches) {
      const shouldActivate = branch.default || 
        await this.evaluateCondition(branch.condition, instance.variables);
      
      if (shouldActivate) {
        hasActivatedBranch = true;
        activatedBranches.push(branch.branchId);
        
        // 找到对应的目标节点
        const edge = definition.edges.find(e => 
          e.source === node.id && e.target === branch.branchId
        );
        
        if (edge) {
          await this.createBranchContext(instance, node.id, branch.branchId);
          await this.advance(instance, branch.branchId);
        }
      }
    }
    
    // 记录激活的分支
    instance.inclusiveActivatedBranches = {
      gatewayId: node.id,
      branches: activatedBranches
    };
  }
  
  // 处理包容汇聚（多对一）
  async handleInclusiveJoin(instance: ProcessInstance, node: ProcessNode): Promise<void> {
    const config = node.config as InclusiveGatewayConfig;
    
    // 获取需要汇聚的分支
    const activatedBranches = instance.inclusiveActivatedBranches?.branches || [];
    const completedBranches = await this.getCompletedBranches(instance, node.id);
    
    // 检查是否所有激活的分支都已完成
    const allCompleted = activatedBranches.every(branchId => 
      completedBranches.includes(branchId)
    );
    
    if (allCompleted) {
      await this.cleanupBranchContexts(instance, node.id);
      
      const definition = await this.getDefinition(instance.definitionId);
      const outgoingEdges = definition.edges.filter(e => e.source === node.id);
      
      for (const edge of outgoingEdges) {
        await this.advance(instance, edge.target);
      }
    }
  }
}
```

### 5.5 流程引擎核心算法

#### 5.5.1 流程推进算法
```typescript
class ProcessEngine {
  // 启动流程实例
  async startInstance(definitionId: string, businessData: any): Promise<ProcessInstance> {
    const definition = await this.getDefinition(definitionId);
    const instance = this.createInstance(definition, businessData);
    
    // 找到开始节点
    const startNode = definition.nodes.find(n => n.type === 'start');
    
    // 推进到第一个任务节点
    await this.advance(instance, startNode.id);
    
    return instance;
  }
  
  // 推进流程
  async advance(instance: ProcessInstance, currentNodeId: string): Promise<void> {
    const definition = await this.getDefinition(instance.definitionId);
    const outgoingEdges = definition.edges.filter(e => e.source === currentNodeId);
    
    for (const edge of outgoingEdges) {
      const targetNode = definition.nodes.find(n => n.id === edge.target);
      
      // 检查条件
      if (await this.evaluateCondition(edge.condition, instance.variables)) {
        await this.enterNode(instance, targetNode);
      }
    }
  }
  
  // 进入节点
  async enterNode(instance: ProcessInstance, node: ProcessNode): Promise<void> {
    switch (node.type) {
      case 'approval':
        await this.createApprovalTask(instance, node);
        break;
      case 'cc':
        await this.sendNotification(instance, node);
        await this.advance(instance, node.id);
        break;
      case 'condition':
        await this.advance(instance, node.id);
        break;
      case 'end':
        await this.completeInstance(instance);
        break;
    }
  }
  
  // 处理审批任务
  async processTask(taskId: string, action: Action, comment?: string): Promise<void> {
    const task = await this.getTask(taskId);
    const instance = await this.getInstance(task.instanceId);
    
    // 更新任务状态
    task.status = 'completed';
    task.action = action;
    task.comment = comment;
    task.completedAt = new Date();
    
    // 根据操作类型处理
    switch (action) {
      case 'agree':
        await this.advance(instance, task.nodeId);
        break;
      case 'reject':
        await this.terminateInstance(instance, 'rejected');
        break;
      case 'return':
        await this.returnToNode(instance, task.nodeId, comment);
        break;
      case 'transfer':
        await this.transferTask(task, comment);
        break;
    }
  }
}
```
