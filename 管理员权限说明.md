# 企业管理系统 - 管理员权限说明

## 一、概述

本文档详细说明企业管理系统中**管理员身份**用户的权限范围、功能模块及操作规范。

---

## 二、管理员角色定义

### 2.1 系统管理员（admin）

| 属性 | 说明 |
|-----|------|
| **角色编码** | admin |
| **角色名称** | 系统管理员 |
| **权限级别** | 超级管理员，拥有系统所有权限 |
| **职责范围** | 系统配置、用户管理、权限分配、数据维护 |

### 2.2 管理员身份识别

系统通过以下条件识别管理员身份：
- 用户角色标识为 `admin`
- 用户名为 `总经理`
- 拥有系统管理模块的访问权限

---

## 三、管理员功能模块权限

### 3.1 系统管理模块（核心权限）

系统管理模块是管理员专属功能区域，包含以下三个子模块：

#### 3.1.1 角色管理

| 功能 | 权限说明 |
|-----|---------|
| 查看角色列表 | 查看系统中所有角色信息 |
| 添加角色 | 创建新角色，设置角色名称、编码、描述、状态 |
| 编辑角色 | 修改已有角色的基本信息 |
| 删除角色 | 删除不再使用的角色 |
| 分配权限 | 为角色分配菜单访问权限 |

**角色属性字段：**
- ID：角色唯一标识
- 角色名称：显示名称（如：技术部经理、财务总监）
- 角色编码：系统识别码（如：tech_manager、cfo）
- 描述：角色职责说明
- 状态：启用/禁用
- 创建时间：角色创建时间戳

#### 3.1.2 菜单管理

| 功能 | 权限说明 |
|-----|---------|
| 查看菜单列表 | 查看系统所有菜单项，支持树形结构展示 |
| 添加菜单 | 创建新菜单项，配置路由路径、组件、图标 |
| 编辑菜单 | 修改菜单名称、路径、排序、状态等 |
| 删除菜单 | 删除菜单项（删除父菜单会级联删除子菜单） |
| 添加子菜单 | 在现有菜单下创建子级菜单 |

**菜单属性字段：**
- 菜单名称：显示在导航中的名称
- 路由路径：前端路由地址（如：/system、/employee-management）
- 组件路径：对应的Vue组件名称
- 图标：菜单图标标识
- 排序：菜单显示顺序
- 上级菜单：父级菜单（顶级菜单为0）
- 状态：启用/禁用

#### 3.1.3 权限分配

| 功能 | 权限说明 |
|-----|---------|
| 选择角色 | 选择需要配置权限的目标角色 |
| 菜单权限配置 | 通过树形控件勾选角色可访问的菜单 |
| 批量操作 | 全部展开、全部收起、全选、清空 |
| 保存配置 | 将权限配置持久化到系统 |

**权限配置规则：**
- 勾选父菜单自动包含所有子菜单权限
- 半选状态表示部分子菜单被选中
- 权限实时生效，无需重新登录

### 3.2 员工管理模块

管理员拥有员工信息的完整管理权限：

| 功能 | 权限说明 |
|-----|---------|
| 查看员工列表 | 查看所有员工的基本信息 |
| 搜索筛选 | 按姓名、部门筛选员工 |
| 添加员工 | 录入新员工信息，包括姓名、部门、职位、联系方式、入职日期、系统角色等 |
| 编辑员工 | 修改员工的基本信息和部门归属 |
| 删除员工 | 从系统中移除员工账号 |
| 重置密码 | 为员工重置登录密码 |

**员工信息字段：**
- 姓名
- 部门（技术部/销售部/财务部/人力资源部/管理部门）
- 职位（根据部门动态显示可选职位）
- 邮箱
- 电话
- 入职日期
- 系统角色（关联权限角色）

**部门与职位映射：**
| 部门 | 可选职位 |
|-----|---------|
| 销售部 | 销售部经理、销售 |
| 技术部 | 技术部经理、项目经理、系统运维工程师、软件研发工程师 |
| 财务部 | 财务总监 |
| 人力资源部 | 人事经理 |
| 管理部门 | 总经理 |

### 3.3 OA审批管理权限

管理员在OA审批流程中具有特殊权限：

| 操作 | 普通用户 | 审批人 | 管理员 |
|-----|---------|-------|-------|
| 提交申请 | ✓ | ✓ | ✓ |
| 查看进度 | ✓ | ✓ | ✓ |
| 撤回申请 | ✓（未审批时） | ✗ | ✓ |
| 同意/驳回 | ✗ | ✓ | ✗ |
| 转交他人 | ✗ | ✓ | ✗ |
| 加签审批 | ✗ | ✓ | ✗ |
| 退回修改 | ✗ | ✓ | ✓ |
| 流程催办 | ✓ | ✓ | ✓ |
| **流程终止** | **✗** | **✗** | **✓** |
| **数据导出** | **✗** | **✗** | **✓** |

**管理员特殊权限说明：**
- **流程终止**：可强制终止任何进行中的审批流程
- **数据导出**：可导出全部审批记录用于审计分析
- **退回修改**：可将审批退回给申请人修改

## 四、审批流程中的管理员角色

### 4.1 请假审批流程

| 审批节点 | 审批人 | 审批条件 |
|---------|-------|---------|
| 申请提交 | 申请人 | 所有员工 |

## 六、权限配置最佳实践

### 6.1 角色创建建议

基于现有组织架构，建议创建以下角色：

| 角色编码 | 角色名称 | 权限范围 |
|---------|---------|---------|
| admin | 系统管理员 | 所有权限 |
| tech_manager | 技术部经理 | 技术部审批 + 员工管理 |
| sales_manager | 销售部经理 | 销售部审批 + 销售数据 |
| cfo | 财务总监 | 财务审批 + 财务报表 |
| hr_manager | 人事经理 | 人事审批 + 考勤管理 |
| ceo | 总经理 | 最终审批 + 全部数据查看 |
| employee | 普通员工 | 基础操作权限 |

### 6.2 权限分配原则

1. **最小权限原则**：只授予完成工作所必需的权限
2. **职责分离原则**：审批权限与系统管理权限分离
3. **定期审计原则**：定期检查权限配置的合理性
4. **离职即回收**：员工离职时立即回收所有权限

## 七、管理员操作规范

### 7.1 账号安全

- 管理员账号密码必须定期更换（建议90天）
- 禁止使用弱密码
- 管理员操作需在安全网络环境下进行
- 发现异常登录立即修改密码并检查日志

### 7.2 操作规范

- 重要操作（如删除角色、删除员工）需二次确认
- 批量操作前建议先备份数据
- 权限变更需记录变更原因
- 系统配置修改需在测试环境验证后再应用到生产环境

### 7.3 数据保护

- 员工个人信息严格保密
- 审批数据不得随意导出外传
- 定期备份系统数据
- 离职员工数据按公司规定保留期限后归档
