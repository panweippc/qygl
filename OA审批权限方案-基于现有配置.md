## 基于现有系统配置的审批权限设计

## 一、现有系统组织架构分析

### 1.1 部门结构

根据系统现有配置，组织架构包含以下5个部门：

| 部门名称 | 部门职责 | 现有职位 |
|---------|---------|---------|
| **技术部** | 产品研发、技术支持、系统运维 | 技术部经理、项目经理、系统运维工程师、软件研发工程师 |
| **销售部** | 客户开发、销售业务、市场拓展 | 销售部经理、销售 |
| **财务部** | 财务管理、资金审批、报销审核 | 财务总监 |
| **人力资源部** | 人事管理、招聘培训、考勤管理 | 人事经理 |
| **管理部门** | 公司管理、战略决策 | 总经理 |

### 1.2 角色权限体系

系统现有角色权限配置：

| 角色编码 | 角色名称 | 权限范围 |
|---------|---------|---------|
| admin | 系统管理员 | 系统超级管理员，拥有所有权限 |
| employee | 普通员工 | 基础操作权限 |

## 二、OA审批角色设计

### 2.1 审批角色定义

基于现有部门和职位，定义以下审批角色：

| 审批角色 | 对应部门/职位 | 审批权限级别 | 主要职责 |
|---------|-------------|-------------|---------|
| **申请人** | 全体员工 | 无 | 发起审批申请、查看进度 |
| **直接上级** | 各部门经理/主管 | 一级审批 | 初审、真实性审核 |
| **部门负责人** | 技术部经理、销售部经理 | 二级审批 | 部门内审批、预算审核 |
| **财务审批人** | 财务总监 | 三级审批 | 费用审核、票据合规 |
| **人事审批人** | 人事经理 | 三级审批 | 人事相关审批 |
| **总经理** | 管理部门/总经理 | 最终审批 | 重大事项、高额审批 |

## 三、审批流程配置

### 3.1 请假审批流程

| 审批节点 | 审批人 | 审批条件 | 操作权限 |
|---------|-------|---------|---------|
| 申请提交 | 申请人 | 所有员工 | 填写表单、上传附件 |
| 一级审批 | 直接上级 | 所有请假 | 同意/驳回/转交 |
| 二级审批 | 部门负责人 | 3天及以上 | 同意/驳回/退回 |
| 三级审批 | 人事经理 | 7天及以上 | 同意/驳回/备案 |
| 最终审批 | 总经理 | 15天及以上 | 同意/驳回 |

**特殊规则：**
- 病假需提供医院证明（附件上传）
- 年假需提前3天申请
- 紧急请假可先口头申请，24小时内补单

### 3.2 报销审批流程

| 审批节点 | 审批人 | 金额条件 | 审核要点 |
|---------|-------|---------|---------|
| 申请提交 | 申请人 | 所有金额 | 填写报销单、上传票据 |
| 一级审批 | 直接上级 | 0-1000元 | 费用真实性 |
| 二级审批 | 部门负责人 | 1000-5000元 | 部门预算审核 |
| 三级审批 | 财务总监 | 5000-10000元 | 票据合规性、预算额度 |
| 最终审批 | 总经理 | 10000元以上 | 大额费用审批 |

**特殊规则：**
- 差旅费需附出差申请单
- 招待费需附招待清单
- 发票需真实有效，抬头正确

### 3.3 采购审批流程

| 审批节点 | 审批人 | 金额条件 | 审核要点 |
|---------|-------|---------|---------|
| 申请提交 | 申请人 | 所有金额 | 填写采购申请 |
| 需求审核 | 部门负责人 | 所有金额 | 采购必要性 |
| 预算审核 | 财务总监 | 2000元以上 | 预算额度审核 |
| 比价审核 | 技术部经理 | 5000元以上 | 技术评估 |
| 最终审批 | 总经理 | 10000元以上 | 大额采购审批 |

**特殊规则：**
- 固定资产采购需走专项流程
- 紧急采购可先执行后补单（限2000元以下）

### 3.4 出差审批流程

| 审批节点 | 审批人 | 出差条件 | 审核要点 |
|---------|-------|---------|---------|
| 申请提交 | 申请人 | 所有出差 | 填写出差计划 |
| 一级审批 | 直接上级 | 国内3天以内 | 工作安排审核 |
| 二级审批 | 部门负责人 | 国内3-7天 | 部门工作协调 |
| 三级审批 | 总经理 | 国内7天以上/国外 | 重要出差审批 |
| 备案 | 人事经理 | 所有出差 | 考勤记录备案 |

### 3.5 办公用品申请流程

| 审批节点 | 审批人 | 金额条件 | 审核要点 |
|---------|-------|---------|---------|
| 申请提交 | 申请人 | 所有金额 | 填写申请单 |
| 一级审批 | 直接上级 | 200元以下 | 需求合理性 |
| 二级审批 | 部门负责人 | 200-1000元 | 部门预算审核 |
| 最终审批 | 总经理 | 1000元以上 | 大额用品审批 |

## 七、系统实现建议

### 7.1 数据库表设计

```sql
-- 审批流程定义表
CREATE TABLE approval_flows (
  id INT AUTO_INCREMENT PRIMARY KEY,
  flowCode VARCHAR(255) NOT NULL COMMENT '流程编码',
  flowName VARCHAR(255) NOT NULL COMMENT '流程名称',
  flowType VARCHAR(255) NOT NULL COMMENT '流程类型(leave/reimburse/purchase等)',
  status VARCHAR(50) DEFAULT '启用' COMMENT '状态',
  createdAt DATETIME NOT NULL,
  updatedAt DATETIME NOT NULL
);

-- 审批节点定义表
CREATE TABLE approval_nodes (
  id INT AUTO_INCREMENT PRIMARY KEY,
  flowId INT NOT NULL COMMENT '流程ID',
  nodeCode VARCHAR(255) NOT NULL COMMENT '节点编码',
  nodeName VARCHAR(255) NOT NULL COMMENT '节点名称',
  nodeOrder INT NOT NULL COMMENT '节点顺序',
  approverType VARCHAR(255) COMMENT '审批人类型(role/dept/position/user)',
  approverValue VARCHAR(255) COMMENT '审批人值',
  conditionExpression TEXT COMMENT '条件表达式',
  createdAt DATETIME NOT NULL
);

-- 审批实例表
CREATE TABLE approval_instances (
  id INT AUTO_INCREMENT PRIMARY KEY,
  flowId INT NOT NULL COMMENT '流程ID',
  applicantId INT NOT NULL COMMENT '申请人ID',
  applicantName VARCHAR(255) NOT NULL COMMENT '申请人姓名',
  applicantDept VARCHAR(255) COMMENT '申请人部门',
  businessData JSON COMMENT '业务数据',
  currentNodeId INT COMMENT '当前节点ID',
  status VARCHAR(50) DEFAULT '审批中' COMMENT '状态',
  createdAt DATETIME NOT NULL,
  completedAt DATETIME COMMENT '完成时间'
);

-- 审批历史记录表
CREATE TABLE approval_history (
  id INT AUTO_INCREMENT PRIMARY KEY,
  instanceId INT NOT NULL COMMENT '实例ID',
  nodeId INT NOT NULL COMMENT '节点ID',
  approverId INT NOT NULL COMMENT '审批人ID',
  approverName VARCHAR(255) NOT NULL COMMENT '审批人姓名',
  action VARCHAR(50) NOT NULL COMMENT '操作(agree/reject/transfer/return)',
  comment TEXT COMMENT '审批意见',
  createdAt DATETIME NOT NULL
);
```

### 7.2 审批人自动匹配逻辑

```javascript
// 审批人匹配逻辑示例
function matchApprover(applicant, nodeConfig) {
  const { approverType, approverValue } = nodeConfig;
  
  switch(approverType) {
    case 'direct_superior':
      // 根据申请人查找直接上级
      return findDirectSuperior(applicant.department, applicant.position);
      
    case 'dept_manager':
      // 查找部门负责人
      return findDeptManager(applicant.department);
      
    case 'role':
      // 根据角色查找
      return findUsersByRole(approverValue);
      
    case 'position':
      // 根据职位查找
      return findUsersByPosition(approverValue);
      
    case 'specific_user':
      // 指定用户
      return findUserById(approverValue);
      
    default:
      return null;
  }
}

// 查找直接上级
function findDirectSuperior(department, position) {
  const superiorMap = {
    '技术部': {
      '软件研发工程师': '项目经理',
      '系统运维工程师': '项目经理',
      '项目经理': '技术部经理'
    },
    '销售部': {
      '销售': '销售部经理'
    }
  };
  
  const superiorPosition = superiorMap[department]?.[position];
  return superiorPosition ? findUserByPosition(superiorPosition) : null;
}
```
